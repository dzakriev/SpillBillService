# Чек‑лист верификации перед релизом модели

## 1. Метрики качества

* Accuracy на тестовом сете > 85%.
* Precision, Recall и F1‑score по ключевым классам не ниже заранее определённых порогов.
* ROC-AUC > 0.9 для бинарных моделей, если применимо.
* Сравнение с предыдущей версией модели: новая модель не должна деградировать более чем на 1–2% по основным метрикам.

## 2. Производительность

* Время предсказания на 1000 объектов < 200 ms.
* Максимальное время отклика модели < 50 ms при одиночном запросе.
* Пиковое потребление памяти не превышает заданный лимит.
* Модель корректно работает в многопоточном / высоконагруженном окружении.

## 3. Проверка данных

* Мониторинг дрифта данных для признака X и остальных критичных признаков: PSI < 0.2.
* Проверка корректности предобработки: отсутствуют неожиданные преобразования или пропуски.
* Проверка схемы входных данных: все требуемые признаки присутствуют и соответствуют ожидаемым типам.
* Валидация на реальных свежих данных из продакшена перед релизом.

## 4. Безопасность и стабильность

* Модель корректно обрабатывает пропущенные значения на входе.
* Нет необработанных исключений во время предсказаний.
* Модель устойчива к аномальным входам (очень большие значения, строки вместо чисел и т.д.).
* Проверка на уязвимости: защита от prompt injection, adversarial атак, утечек данных.
